    4. Checks its DNS cache
    5. If the primary DNS server doesn't have the requested domain in its DNS cache, it sends out the request to the root name server.
    6. Root name server returns IP of a name server for requested TLD(in this case the com name server).
    7. Request IP of a name server for that website
    8. .com name server will return the IP address of the DNS server for that website
    9. Request for IP address for that website
    10. Return IP address of web server
    11. Return IP address of that website
    12. The next step is the Browser request page
    13. The last step is to return the requested page
